Version 5 represents a major\nrefactoring of the original Fortran 77 Laura code toward a modular structure\na\xef\xac\x80orded by Fortran 95. The refactoring improved usability and maintainability by eliminating the requirement for problem-dependent recompilations,\nproviding more intuitive distribution of functionality, and simplifying interfaces required for multi-physics coupling. As a result, Laura now shares\ngas-physics modules, MPI modules, and other low-level modules with the\nFun3D unstructured-grid code. In addition to internal refactoring, several\nnew features and capabilities have been added, e.g., a GNU-standard installation process, parallel load balancing, automatic trajectory point The signi\xef\xac\x81cant uncertainty\nin these modeling parameters corresponds to potentially signi\xef\xac\x81cant uncertainty in\nthe simulated radiative heating. This radiative heating parametric uncertainty represents a signi\xef\xac\x81cant fraction of the radiative heating margin applied for vehicle\ndesign, with the rest of the margin based primarily on comparisons with available\nexperimental data (mostly from shock tubes). This radiative heating margin is often\ngreater than 50%, which means it can a have signi\xef\xac\x81cant impact on the heatshield\nsizing. However, it is typically computed (through detailed parametric uncertainty\nanalyses and comparisons with shock tube measurements) for a single vehicle body\npoint (usually the stagnation-point) at perhaps only a single trajectory point (usually peak heating). These limited evaluations are the result of complexity in the\nparametric uncertainty analysis, due to the large number of parameters involved,\nand di\xef\xac\x83culty in relating shock tube measurements to \xef\xac\x82ight vehicle radiative heating.\nTo remove these di\xef\xac\x83culties and enable routine evaluations of the parametric\nuncertainty and shock-tube based components to the radiative heating margin, the\nhara uq code was developed. This code, which is run as a post-processing step to a\nconverged laura solution, performs these analyses automatically. The application of\nhara uq is recommended as the standard approach for radiative heating simulations\nthat support vehicle design projects. It provides both the nominal ray-tracing radiative heating values and the corresponding parametric uncertainty and shock-tube\ninformed bias, both of which represent the dominant contributions to the radiative\nheating margin.\n\nD.1\n\nRunning hara uq\n\nThe hara uq code is designed to require minimal user input. Staring with a converged laura solution that includes coupled radiation (the convergence here refers\nto the radiative heating not changing with further iterations, not necessarily a small\nresidual), the primary task of the user is to decide the number of processors to use.\nThis number should be a multiple of the number of blocks. Because hara uq internally runs simultaneous laura cases, the larger the number of processors results\nin the code running more simultaneous laura cases. In other words, increasing the\nnumber of processors to a large multiple of the number of blocks will not result in\nmultiple processors per block, which may result in performance penalties.\nOnce the number of processors is chosen, hara uq is run identically to laura.\nThe defaults are set so that the code considers radiation spectrum modeling parameters, non-Boltzmann atomic and molecular rates from the strongest radiating atom\nand molecule, and \xef\xac\x82ow\xef\xac\x81eld chemical kinetics in determining the radiative heating\nparametric uncertainty. A sensitivity analysis is \xef\xac\x81rst run for each of these param-\n\n95\n\neters assuming tangent-slab. The results of this sensitivity analysis are saved in\nthe laura sensitivity rad.q and uncertainty matlab out.m \xef\xac\x81les. This sensitivity analysis informs a set of parameter adjustments to determine the parametric\nuncertainty. Ray-tracing is applied by default for this parametric uncertainty computation. Finally, the shock tube informed correction to the radiative heating is\nevaluated using the approach presented by Johnston. Both the parametric uncertainty and the shock-tube informed correction to the radiative heating are automatically added to the laura surface.q and laura blayer.dat \xef\xac\x81les, which are renamed\nlaura surface uq.q and laura blayer uq.dat. The nominal ray-tracing radiative\nheating computed by hara uq is also added to these \xef\xac\x81les.\nThe computational time required by hara uq can be signi\xef\xac\x81cant. It is roughly\nequivalent to 20 to 40 laura solutions run for 5000 \xef\xac\x82ow\xef\xac\x81eld iterations and raytracing performed for all body points de\xef\xac\x81ned by iinc rad and jinc rad in laura namelist data.\nFortunately, these 20 to 40 solutions are embarrassingly parallel, so that increasing\nthe number of processors (in increments of the number of blocks) reduces this time\naccordingly. However, for larger scale problems where su\xef\xac\x83cient processors are not\navailable, many of the default options may be altered to reduce the computational\ntime. This can be done with minimal loss in accuracy assuming some knowledge of\nthe problem of interest. For example, an Earth entry case at a velocity below 9 km/s\nwill experience minimal atomic radiation, and therefore, the atomic radiation modeling parameters may be ignored by hara uq, which reduces the computational time\nsigni\xef\xac\x81cantly. ABSTRACT\n\nThis users manual provides in-depth information concerning installation and execution of L AURA, version 5. L AURA\nis a structured, multiblock, computational aerothermodynamic simulation code. Version 5 represents a major\nrefactoring of the original Fortran 77 L AURA code toward a modular structure a\xef\xac\x80orded by Fortran 95. The\nrefactoring improved usability and maintainability by eliminating the requirement for problem-dependent\nrecompilations, providing more intuitive distribution of functionality, and simplifying interfaces required for\nmulti-physics coupling. As a result, L AURA now shares gas-physics modules, MPI modules, and other low-level\nmodules with the F UN 3D unstructured-grid code. In addition to internal refactoring, several new features and\ncapabilities have been added, e.g., a GNU-standard installation process, parallel load balancing, automatic\ntrajectory point sequencing, free-energy minimization, and coupled ablation and \xef\xac\x82ow\xef\xac\x81eld radiation.\n15.
