With simulation the line can be\nstudied under a controlled environment\nby varying one parameter at a time. In other\nwords,\nsimulation\ncan be considered\nas inexpensive\ninsurance\nagainst\ncostly\nmistakes.\nThere are several drawbacks to using computer simulation.\nOne of the most serious\nis the length of time to develop, verify, and validate the simulation\nmodel.\nQuite\noften management\ncannot wait this length of time for the simulation\nresults.\nA\nsecond drawback\nis that the firm needs someonetrained\nin simulation.\nThis person\nis not only difficult to locate, but generally cannot be justified on a full-time basis.\nWhat\n\nis Computer\n\nSimulation\n\nSimulation\nconsists of developing a representation,\nor model, of a real-world\nsystem\nand then experimenting\nwith the model to study the operation\nof the system over\ntime. Models of very simple systems often can be solved mathematically.\nHowever,\nmost models of complex, real-world\nsystems\ncannot, and instead must be solved\nusing simulation.\nIn these instances, a computer\nmodel of the system is developed.\nThe computer\nmodel is generally\nwritten\nor programmed\nusing commercially\navailable simulation software.\nWith simulation\nit is possible to manipulate\nthe model\nrather than the real-world system.\nIn a manufacturing\nenvironment,\nsuch real-world\nmanipulation\nis often too expensive and impractical,\nopening opportunity\nfor the use\nof simulation.\nFigure 3 outlines the steps in a computer\nsimulation.\nFirst the user must define the\nsystem, or problem.\nQuite often this step is the most difficult; Two approaches used in starting a simulation\nmodel are start the system empty and idle and set the starting conditions as close to\nsteady state mean or mode as possible. The first approach is most commonly used\nin manufacturing systems because of it simplicity. Using this method, all queues, or\nbuffers, start empty and facilities, qr machines, start idle.\nA simulation can be terminated by stopping the creation of new events and then\nallowing the system to return to an empty and idle state. It is important to note that\nincluding the measurements collected after terminating new events also introduces\nbias which can be serious, especially if the total run time is short.\nTwo approaches commonly used in manufacturing models to determine stopping\nconditions are to stop the system after a given amount of production or after a given\ntime. By using either of these approaches, no limit is placed on the number of parts\nentering the system. Therefore, at the completion of the simulation, parts are still in\nthe system and machines are still be utilized.\nSteady State Analysis. Generally there is no single point during the execution of\nthe simulation model beyond which the system is in steady state, or equilibrium.\nTherefore, there is a problem in finding the point which the modeler is willing to\nneglect the error made by considering the system in equilibrium.\nOne common approach in determining equilibrium is to execute the model for a long\nenough period of time to ensure the system\'s performance does not depend on the\nstarting condition of the model. However, in most situations, the error resulting from\nthe initial conditions must be taken into consideration. Several heuristic rules to\nestimate system steady state are Conway rule; modified Conway rule, crossing of\nthe mean rule, cumulative mean rule and deletion rule.\nFor example, Figure 4 is a plot of ten batch means with each batch having a sample\nsize of twenty-five\nobservations.\nThese batches\nare obtained\nby running\nthe\nsimulation\nmodel for a time period sufficient to complete\ntwenty-five\nparts for each\n\n7\n\nbatch. The response variable, such as production per hour, is then measured. The\nmodel continues to run and the response variable is measured after each batch of\ntwenty-five parts.
