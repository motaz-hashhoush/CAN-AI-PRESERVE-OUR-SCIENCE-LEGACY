GDR\nfiles provide WFF with the same products as used by TOPEX scientists and researchers.\nThe flow of altimeter data is depicted in Figure 3-1 "TOPEX ALT Dataflow (GDR\nEmphasized)," where emphasis is placed on the GDR data flow. The TOPEX Ground\nSystem extracts altimeter science and engineering minor frames from the spacecraft\ntelemetry. Common frames are then merged and time-sorted to create Altimeter\nInstrument Files (AIFs).AIFs and the earth-location data are processed using Telemetry and Science-level algorithms to create once-per-pass (about 56 minutes) SDR files.\nThe SDR files are further processed using Geophysical-level algorithms to create\nInterim Geophysical Data Records (I/GDR).The processed IGDRs are placed in temporary storage. Once precision orbit data has been received by JPL, the new orbit\ndata is merged into the IGDRs and final GDRs are created. At the end of a 10-day\nCycle, the GDR pass files are copied to tape.\nFor a complete description of the GDR formats, see TOPEX Ground Svstem Software\nInterface Specification,Vol. 2: Desim (SIS-2). Geophvsical Data Record (GDR) - GDR\nData. Interim Geophvsical Data Record (IGDR) - GDR Data.\n\n3.2\n\nDistribution\n\nIndividual IGDR pass files are made accessible to WFF via the NASA Science Internet\n(NSI) using FlTprotocols. GDR per-cycle exabyte tapes are mailed to GDR data subscribers. The WFF SWDT receives these tapes and archves them for later use. The\nSWDT redistributes copies of the GDR tapes to external users if so requested.\n\n3.3\n\nStorage\n\nA single GDR pass file requires approximately 500 kilobytes of disk storage.
